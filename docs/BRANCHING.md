# Соглашения по веткам и коммитам

## Модель ветвления

Проект использует GitHub Flow с дополнительными правилами для практических работ:

### Основные ветки
- `main` — стабильная ветка, защищена от прямых пушей
- `p<nn>-<topic>` — ветки для практических работ (например, `p02-git-workflow-setup`)

### Правила именования других веток
- **Фичи**: `feat/<kebab-case-description>`
- **Багфиксы**: `fix/<kebab-case-description>`
- **Рефакторинг**: `refactor/<kebab-case-description>`

## Стиль коммитов

Используем [Conventional Commits](https://www.conventionalcommits.org/):

### Типы коммитов
- `feat:` — новая функциональность
- `fix:` — исправление бага
- `refactor:` — рефакторинг кода
- `test:` — добавление или изменение тестов
- `docs:` — обновление документации
- `chore:` — вспомогательные изменения (зависимости, конфиг)
- `ci:` — изменения CI/CD конфигурации
- `security:` — исправления безопасности
